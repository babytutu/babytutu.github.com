插件设置plugins

只简单说下几个常用的插件

  • html-webpack-plugin 自动化创建html文件
  • clean-webpack-plugin 清理打包目录，这个非必须，其实就是在rimraf基础上包装了一下，可以删除多个打包目录
  • uglifyjs-webpack-plugin webpack4开始mode为production默认开启，无需单独设置
  • mini-css-extract-plugin 分离css文件，之前的extract-text-webpack-plugin暂不支持webpack4，很多bug，作者好像已经被铺天盖地的issues吓傻了
  • optimize-css-assets-webpack-plugin 压缩css，已经支持4了，很神速啊
  • webpack.optimize.MinChunkSizePlugin 通过合并小于 minChunkSize 大小的 chunk，将 chunk 体积保持在指定大小限制以上。

加入eslint-loader

初始化eslint配置

根据问题简单选择就好了, 我选了Airbnb，然后自动给装了2个插件并自动生成了.eslintrc.js

webpack/webpack.common.js

启动开发模式npm start,就会发现打印信息里很多错误，因为示例代码都是随便写写的啦，好吧，提示说可以用--fix来修复部分错误，那就再加个修复脚本

运行npm run lint-fix，发现大部分问题修复了，然而还有1个错误是多余的后缀名Unexpected use of file extension "js" for "./print.js",把.js去掉就好啦，还有1个no-console的警告，这个问题不大，平时调试时候用到，记得要去掉
